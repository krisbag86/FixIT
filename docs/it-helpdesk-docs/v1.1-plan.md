# FixIT v1.1 - plan prac

`main` pozostaje wersja 1.0.
Branch roboczy: `v-1.1`.

Wersje 1.1 wydajemy, gdy ta lista bedzie miala lacznie 10 wykonanych zmian.

## Checklist zmian

- [x] 1. Ikona w pasku przegladarki
  - Dodano `app/icon.svg`.
  - Podpieto ikone w metadata aplikacji.
  - Zweryfikowano `typecheck` i `build`.

- [ ] 2. Lepsza lista zgloszen
  - Filtry: status, priorytet, sklep, kategoria, moje/przypisane.
  - Wyszukiwarka po numerze, tytule i opisie.

- [ ] 3. Widok "Moje zadania" dla IT
  - Lista ticketow przypisanych do zalogowanego agenta.
  - Licznik otwartych zadan.

- [ ] 4. Czytelna os czasu zgloszenia
  - Historia zdarzen w szczegolach ticketu.
  - Status, priorytet, przypisanie, komentarze, zalaczniki.

- [ ] 5. Szybkie odpowiedzi IT
  - Szablony odpowiedzi dla czestych problemow.
  - Agent moze wybrac szablon i edytowac go przed wyslaniem.

- [ ] 6. Potwierdzanie rozwiazania przez zglaszajacego
  - Reporter moze zamknac zgloszenie po statusie `RESOLVED`.
  - Potwierdzenie zmienia status na `CLOSED`.

- [ ] 7. Widoczne SLA na listach i szczegolach
  - Termin SLA przy tickecie.
  - Oznaczenia: w terminie, blisko terminu, po terminie.

- [ ] 8. Usprawnienie formularza nowego zgloszenia
  - Po wyborze sklepu pokazac nazwe, adres i mail.
  - Poprawic wygode formularza na telefonie.

- [ ] 9. Ulepszenia Bazy Wiedzy
  - Wyroznic artykul `Kontakty`.
  - Ulepszyc wyszukiwanie po tresci, miescie, kodzie sklepu i mailu.

- [ ] 10. Checklist przed wydaniem 1.1
  - `npm run typecheck`
  - `npm run build`
  - testy/regresja najwazniejszych sciezek
  - smoke test Railway
  - aktualizacja changelogu
  - merge `v-1.1` do `main`

## Zasady pracy

- Kazda zmiana dostaje osobny commit na branchu `v-1.1`.
- Po wykonaniu zadania odznaczamy je tutaj.
- Do `v1.1-changelog.md` dopisujemy krotka notke o wykonanej zmianie.
- Merge do `main` dopiero po zamknieciu 10 punktow.
